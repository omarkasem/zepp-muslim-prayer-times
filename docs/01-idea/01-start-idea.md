I have a software/plugin/app idea.

Help me refine it. Focus on:
- the real pain point
- who would actually use this
- why they would pay for it
- what makes it different
- what should NOT be in V1
- realistic implementation scope

Be brutally honest. If the idea is weak, saturated, or not worth building, say so now — this is the cheapest moment to kill it.

When refinement is complete, generate the final file with this structure:

# Product Description
# Problem Statement
# Target Users
# Main Features
# MVP Features
# Future Features
# Risks
# Possible Competitive Advantages
# Honest Assessment
- One paragraph: should I keep going, adjust, or drop this?

Start by asking me about the idea.
